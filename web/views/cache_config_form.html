Configuração da Cache
bits

Aqui você determina quantos bits são necessários para representar o endereço da memória física. Você pode colocar entre 16 bits (2^16 bytes de endereçamento) e 128 bits (2^128 bytes de endereçamento disponíveis).

bits

Aqui você determina quantos bits são necessários para representar o offset de cada bloco da cache. Você só pode colocar entre 2 bits (4 bytes cada bloco) de offset e 13 bits (8192 bytes cada bloco) de offset.

bits

Aqui você determina quantos bits são necessários para representar a faixa de índices (conjuntos, ou linhas da cache, ou entradas da cache, no caso de Fully-Associative não tem índices por isso existe 0 de cota inferior). Você só pode colocar entre 0 (fully-associative) e 128 bits (2^128 entradas de cache) como tamanho do campo de índice.

bits

Aqui você determina quantos bits são necessários para representar a tag. No mínimo 8 bits e no máximo 120. Pois nessa aplicação você tem um mínimo de 8 bits de offset. Existem outros tipos de cache com tamanhos diferentes.

ways

Aqui você determina o número de ways, Diretamente Mapeada é 1-way, e aí tem 2-way, 3, 4, 5, ... até completamente associativa em que você deve colocar 0 em índice e colocar aqui a quantidade de ways. Completamente associativa tem 1 conjunto, porém muitos tag+bloco no mesmo conjunto.
